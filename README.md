# Zodiac OS Constellation Template

A ready-to-use starting point for managing Zodiac account constellations as code.

## Getting started

1. Generate a Zodiac OS API key at [app.zodiac.eco/admin/api-keys](https://app.zodiac.eco/admin/api-keys)
2. Copy `.env.template` to `.env` and fill in your credentials
3. `bun install`
4. `bun pull` ‚Äî fetches your org data (users, vaults) and contract ABIs, generates typed codegen
5. Edit `constellation/index.ts` to define your constellation
6. `bun apply` to deploy

## Workflow

### 1. Define account structure ‚Äî `constellation/index.ts`

Declare Safes, roles mods, and delay mods using the `constellation()` entrypoint. Existing accounts from your Zodiac OS workspace are available via bracket access (`eth.safe["GG DAO"]`); new accounts are created by invoking the bracket accessor with a fresh label (`eth.safe["My New Safe"]({ nonce, threshold, owners, ‚Ä¶ })`). Only nodes you `export` end up in the constellation.

New nodes must provide every mandatory field ‚Äî `nonce`, `threshold`, `owners` for Safes and `nonce` for Roles mods. The SDK does not fill in runtime defaults, so a missing required field is caught by TypeScript before you apply. Existing-account overrides are all optional.

See `constellation/sketch.ts` for a fuller set of patterns (existing vs new accounts, forward refs, etc.).

### 2. Define role members ‚Äî `constellation/roles/<role>/members.ts`

Each role directory exports an array of member addresses:

```ts
export default [
  "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
] satisfies `0x${string}`[];
```

### 3. Define role permissions ‚Äî `constellation/roles/<role>/permissions.ts`

Collect the contract addresses your role interacts with in `contracts.ts`, then express permissions via the `allow` kit (or `defi-kit` for preset actions):

```ts
export default [
  allow.arb1.weth.deposit({ send: true }),
  allow.arb1.weth.withdraw(),
] satisfies Permissions;
```

## Codegen ‚Äî `zodiac.config.ts` + `bun pull`

The project integrates the [`@zodiac-os/sdk`](https://github.com/gnosisguild/zodiac-os-sdk) CLI for codegen. Configuration lives in `zodiac.config.ts`, which wires the API key (from `.env`), re-exports `contracts.ts`, and points to the ABIs directory:

```ts
import { defineConfig } from "@zodiac-os/sdk/cli/config";
import contracts from "./contracts";

export default defineConfig({
  apiKey: process.env.ZODIAC_API_KEY as `zodiac_${string}`,
  contracts,
  abisDir: "abis", // default ‚Äî change if you want ABIs stored elsewhere
});
```

Running `bun pull` does two things:

- **`pull-org`** ‚Äî fetches the users and vaults from your Zodiac OS workspace and emits typed data under `node_modules/.zodiac-os/`. This is what powers bracket-access autocompletion (`eth.safe["..."]`, `eth.user["..."]`).
- **`pull-contracts`** ‚Äî fetches ABIs for every address in `contracts.ts`, saves them to `abisDir` (default `./abis/`), and generates `.zodiac-os/allow.d.ts` so `allow.<chain>.<contract>.<fn>` autocompletes with named parameters.

Run the halves individually with `bun pull-org` / `bun pull-contracts` when iterating.

> **VS Code tip:** after codegen, restart the TypeScript server (`Cmd+Shift+P` ‚Üí "TypeScript: Restart TS Server") if autocomplete doesn't reflect your changes.

## Managing contract addresses

All contract addresses live in `contracts.ts`, keyed by chain prefix (`eth`, `arb1`, `gno`, `megaeth`, ‚Ä¶) and a nested path:

```ts
export default {
  eth: {
    weth: "0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2",
    uniswap: {
      positions_nft: "0xC36442b4a4522E871399CD717aBDD847Ab11FE88",
    },
  },
  megaeth: {
    CompositeExchange: "0x5e3Ae52EbA0F9740364Bd5dd39738e1336086A8b",
  },
} satisfies ContractsConfig;
```

Every time you add, remove, or rename an entry, **re-run `bun pull-contracts`**. If a contract isn't verified and its ABI can't be fetched, the CLI prints the file path where to paste the ABI JSON manually, then re-run.

## Commands

| Command | What it does |
| --- | --- |
| `bun pull` | Fetch org data + contract ABIs and regenerate all codegen |
| `bun pull-org` | Fetch only users/vaults from your workspace |
| `bun pull-contracts` | Fetch only contract ABIs and regenerate the `allow` kit types |
| `bun apply` | Deploy the constellation defined in `constellation/index.ts` |
| `bun format` | Prettier across the repo |
